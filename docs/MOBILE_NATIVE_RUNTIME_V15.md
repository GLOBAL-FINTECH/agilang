# AGILANG v1.5 Mobile Native Runtime Bridge

AGILANG v1.5 extends the mobile story beyond React Native scaffolding by adding Android and iOS native runtime bridge targets.

## Mobile support model

AGILANG mobile development has two layers:

1. **React Native / Expo application layer** — generated by `agi react mobile <name>`.
2. **Optional native C runtime bridge** — generated by `agi mobile native-bridge <name>`.

The React Native app works without the native bridge. The bridge is used when you want to link AGILANG's C HTTP/WebSocket runtime into an Android/iOS build for native transport acceleration and device-local runtime capability checks.

## Commands

```bash
agi mobile capabilities
agi mobile platform-matrix
agi mobile doctor
agi mobile native-bridge test-app-two-native --target both
```

## Supported mobile targets

| Target | Runtime artifact | Build toolchain |
|---|---|---|
| Android arm64-v8a | `libagilang_net_runtime.so` | Android NDK + CMake |
| Android x86_64 | `libagilang_net_runtime.so` | Android NDK + CMake |
| iOS arm64 device | `libagilang_net_runtime.a` | Xcode clang |
| iOS Simulator arm64 | `libagilang_net_runtime.a` | Xcode clang |
| iOS Simulator x86_64 | `libagilang_net_runtime.a` | Xcode clang |

## Important production note

Android and iOS runtime artifacts must be built by the Android/iOS toolchains. The package includes source, bridge wrappers, manifests, and CI/release wiring. It does not fake mobile binaries when built on Linux.

## Full app flow

```bash
agi new test app two
cd test-app-two
agi run
agi serve src/main.agi --host 127.0.0.1 --port 8000
agi run src/realtime.agi
```

Generate mobile clients:

```bash
agi react mobile test-app-two-mobile
agi mobile native-bridge test-app-two-native --target both
```

The generated mobile client consumes the AGILANG backend over WebSocket/WebRTC signaling. The native bridge can be linked later when you build an Android/iOS native runtime package.
