# Customization Guide

This starter separates application logic from user-editable presentation files.

## Homepage

The homepage is controlled by two files:

```text
src/main.agi
resources/views/index.ags
```

`src/main.agi` registers the homepage route:

```agilang
app.get("/", home)
```

The `home` handler renders the editable template through `index_view(user)`, and `index_view` loads:

```text
resources/views/index.ags
```

Edit this file to change the hero headline, reactive stats, buttons, and homepage copy. The `/home` route uses `resources/views/home.ags`, and the EVM page uses `resources/views/evm.ags`.

## Layout

All pages are wrapped by:

```text
resources/views/layout.ags
```

Edit it to change the header, brand mark, navigation position, global meta tags, or main page shell.

Navigation is separated into:

```text
resources/views/partials/nav_guest.ags
resources/views/partials/nav_user.ags
```

Use `nav_guest.ags` for visitors and `nav_user.ags` for signed-in users.

## Styles

Main styles live in:

```text
resources/assets/css/app.css
resources/assets/css/account.css
```

Use `app.css` for global layout, homepage, dashboard, buttons, cards, and responsive behavior. Use `account.css` for profile/security/account pages.

## Dashboard

The dashboard page template is:

```text
resources/views/dashboard.ags
```

Reusable dashboard pieces live in:

```text
resources/views/components/item_row.ags
resources/views/components/admin_users.ags
resources/views/components/user_row.ags
resources/views/components/empty_state.ags
```

## Add a new page

1. Create a new template:

```text
resources/views/pricing.ags
```

2. Add a view function in `src/main.agi`:

```agilang
fn pricing_view():
    return render_ags("../resources/views/pricing.ags", {})["body"]
```

3. Add a route handler inside `create_app()`:

```agilang
fn pricing(request):
    let user = current_account(request)
    return html_response(layout("Pricing", pricing_view(), user, "pricing"))
```

4. Register the route:

```agilang
app.get("/pricing", pricing)
```

5. Add a navigation link in one of these files:

```text
resources/views/partials/nav_guest.ags
resources/views/partials/nav_user.ags
```

## Template variables

Templates use `{{ name }}` for escaped values and `{{{ body }}}` for trusted HTML fragments.

Examples:

```html
<h1>{{ title }}</h1>
<div>{{{ rows }}}</div>
```

Use triple braces only for HTML generated by your own app functions.
